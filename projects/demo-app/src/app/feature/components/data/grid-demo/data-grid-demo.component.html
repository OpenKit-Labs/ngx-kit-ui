Data Grid A fully-featured data grid with viewport-aware height modes, sortable and searchable columns, a composable renderer system, and a first-class in-memory data source for quick prototyping. Usage Import the module Or import the standalone component directly Then add it to your template Component Inputs KitDataGridColumnConfig Each entry in config.columns is a KitDataGridColumnConfig<T> object. Dot-notation paths let you reach nested fields (e.g. location.city). KitDataGridConfig Pass a KitDataGridConfig object to the [config] input to control grid-level behaviour. KitDataGridHeightConfig KitDataGridHeightConfig controls how the grid fills vertical space. Use 'flex' (default), 'viewport', 'auto', or a pixel number. KitDataGridRowConfig Set via config.row. All properties are optional; combine minHeight and maxHeight to let rows adapt to their content within a range. Data Source The grid relies on an abstract KitDataGridDataSource<T>. Implement init(query) (called once on mount) and queryChange(query) (called on every sort, filter, or page change) — both must return a Promise<KitDataGridResult<T>>. For local arrays, use the built-in KitDataGridInMemoryDataSource. It supports multi-column sorting, contains-string filtering, and pagination out of the box. Built-in Renderers Import the KitDataGridBuiltinCellRenderers, KitDataGridBuiltinHeaderRenderers, and KitDataGridBuiltinFooterRenderers namespaces to access pre-built renderer descriptors. Control Header — Config KitDataGridBuiltinHeaderRenderers.Control(config) adds sort and search toggles to a column header. Default Footer — Config KitDataGridBuiltinFooterRenderers.Default(config) renders a pagination bar. Custom Renderers Any standalone Angular component can act as a renderer. Implement the corresponding interface, then wrap it with the matching factory helper. Viewport Height + Sortable & Searchable Columns The grid fills from its top edge to the bottom of the browser window. The Joined column uses TimeAgo cell rendering and a sort-only control header. The City column supports both sort and search. Fixed Height (400 px) Grid locked to an explicit pixel height with internal scrolling. No Footer Set footer.isVisible: false to hide the footer while keeping the renderer configured. Custom Cell Renderer with Config The Tags column renders through a custom CustomButtonCellComponent that accepts a color config.
