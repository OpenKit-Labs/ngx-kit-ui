Page The page component provides the structural chrome for a full screen: a sticky header for the top bar, a sticky footer for the bottom bar, a scrollable content area, and an optional pull-to-refresh mechanism. It should be used as a direct child of kit-app-root for every full-screen view. For responsive width constraints, pair it with kit-container. Usage Import the layout module Or you can import the single component Use kit-page together with kit-container for a full layout with breakpoint width constraints. Inputs Outputs Content Slots Pull-to-Refresh Set [pullDownThreshold] to a positive pixel value to enable pull-to-refresh. The refreshTriggered output fires when the user pulls beyond the threshold. No-Scroll Mode Enable [noScroll] for full-screen layouts where content should fill the viewport exactly (e.g. login pages, splash screens).
